# 要件定義書：フェーズ3 収益最大化・生存維持基盤

## 1. 目的

獲得したユーザーの「継続性」を科学し、事業の LTV（顧客生涯価値）を最大化する。
一時的な売上ではなく、安定したキャッシュフローを維持するための「防衛線」を可視化し、コンシェルジュに対して「持続可能な起業モデル」であることを証明する。

---

## 2. 収集対象指標（KPI定義）

フェーズ2までの指標を基礎とし、収益性と継続性に完全にフォーカスした指標を追加する。

| 指標名 | 取得元 | ビジネス上の役割 |
| --- | --- | --- |
| **MRR（月次経常収益）** | Stripe API | 目標10万円に対する現在地と、事業の「定常的な筋力」。 |
| **チャーンレート（解約率）** | Stripe API | **最重要**。延命術が顧客に受け入れられているかの究極の審判。 |
| **LTV（顧客生涯価値）** | 内部計算（MRR / チャーン） | 1人の顧客獲得にどれだけの労力を割けるかの基準。 |
| **ユニットエコノミクス** | 統合データ（LTV / CPA） | 事業の健全性。LTV &gt; CPA（顧客獲得コスト）が維持されているか。 |
| **ARPU（ユーザー平均単価）** | Stripe API | 付加価値向上（単価アップ）の余地を測る。 |

---

## 3. 実装要件

### 3.1 取得手法

- **決済API連携**: `src/utils/stripe-client.ts` を新規作成し、Stripe API から契約日・解約日・売上データを取得
- **コホート分析**: 「○月に加入したユーザー」が何ヶ月後に何％残っているかを追跡する計算ロジックをローカルで実装
- **認証**: Stripe シークレットキーを `.env` で管理

```sh
# .env に追加する環境変数（フェーズ3）
STRIPE_SECRET_KEY=...
```

### 3.2 集計プロセス

1. **財務データ取得（Fetch）**: `src/utils/stripe-client.ts` で Stripe API を呼び出し、MRR・解約率・ARPU を算出
2. **相関分析（Analysis）**: 「滞在時間の長いユーザー」と「解約率」の相関をローカル計算で分析
3. **予測（Forecast）**: 現在の成長率とチャーンレートから3〜6ヶ月後の収益予測を算出
4. **HTML生成（Render）**: フェーズ1・2の形式を維持しつつ、財務ダッシュボードセクションを追加

### 3.3 型定義の拡張

`src/types/index.ts` の `ReportData.stats` に以下を追加する。

```typescript
revenue: {
  mrr: number;             // 月次経常収益（Stripe API）
  churnRate: string;       // チャーンレート（Stripe API）
  ltv: number;             // 顧客生涯価値（内部計算: ARPU / churnRate）
  arpu: number;            // ユーザー平均単価（Stripe API）
  unitEconomics: string;   // LTV / CPA の比率（内部計算）
};
```

> **注意**: `unitEconomics` の算出には CPA（顧客獲得コスト）が必要。CPA は広告費等の外部コストを含むため、初期は手動入力値を `.env` で設定する運用とする。

---

## 4. レポート構成（HTML出力仕様）

「経営者」としてコンシェルジュと対等に議論するための、財務的説得力を持たせた構成とする。
フェーズ1・2のセクションはそのまま維持し、以下を追加する。

- **収益ダッシュボード**: MRR・総会員数・前月比成長率（MoM）を最上部に配置
- **生存曲線（Retention Curve）**: ユーザーが時間とともに離脱していく割合を可視化
- **解約理由サマリー**: 解約時に取得したアンケートや行動ログから、構造的な不満を特定
- **損益分岐点チャート**: サーバー代等の固定費と、目標利益10万円までの距離を可視化

---

## 5. 運用要件

- **実行頻度**: 月次（月末または月初）
- **実行コマンド**: `npm run report`（フェーズ1・2と同一）

### 判断基準（ネクストアクション）

| 観測結果 | 判断・対処 |
| --- | --- |
| チャーンレートが想定（例: 5%）を超えた場合 | 新規獲得を止め、既存コンテンツの「延命（質向上）」にリソースを全振りする |
| LTV が順調に伸びている場合 | 副層（非技術者）向け有料オプションや高単価な個別支援プランの検討を開始 |
| MRR 10万円達成時 | 「防衛拠点」化が完了したと見なし、次のプロダクト展開や自動化による不労化をコンシェルジュと相談する |
