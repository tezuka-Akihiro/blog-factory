# Phase 2: リファクター実行 or 計画書生成

## AI役割定義

あなたは**コードクオリティエンジニア**です。
選択された区分の対象ファイルを精査し、作業量を判定した上でリファクターを実行、または並列実行計画書を生成してください。

## 思考プロセス（CoT）

1. **スコープ確認**: 選択区分の `targets` と `refactor_guide` を `docs/categories.yaml` から読み込む
2. **ファイル列挙**: Glob で対象ファイルを列挙し、件数・行数を把握する
3. **作業量判定**: 判定基準に照らしてモードを決定する
4. **実行 or 生成**: 判定結果に従って処理を進める

## 実行手順

### Step 1: 区分情報を読み込む

```text
.claude/skills/auto-refactor/docs/categories.yaml
```

選択区分の `targets` と `refactor_guide` を確認する。

### Step 2: 対象ファイルを列挙する

`targets` ディレクトリ配下のファイルを Glob で列挙し、ファイル数と各ファイルの行数を確認する。

### Step 3: 作業量を判定する

| 条件 | モード |
| :--- | :--- |
| 対象ファイルが **10ファイル以下** かつ 各ファイルが **300行以下** | 実行モード |
| 上記を超える場合 | 計画書生成モード |

> この基準はコンテキスト汚染防止のための目安。内容を精査した上で最終判断する。
> 明らかに軽微な区分（spec-yaml 等）は柔軟に実行モードを選択してよい。

---

## 実行モード

`docs/categories.yaml` の当該区分 `refactor_guide` に記載された方針に従ってリファクターを実施する。

専用スキル（`skill` フィールド）が定義されている場合はそれを優先する。

実行後は `prompts/03-record.md` を参照して記録を更新する。

## Output形式（実行モード）

```text
## リファクター実施内容

区分: {name}（{id}）

### 変更ファイル一覧
- {ファイルパス}: {変更内容の概要}

### 変更サマリー
{実施したリファクターの概要}

→ prompts/03-record.md に進みます。
```
