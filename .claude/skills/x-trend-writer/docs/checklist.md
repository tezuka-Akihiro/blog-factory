# 公開前チェックリスト

記事を完成させる前に、以下のすべてを確認してください。

---

## ✅ 情報の正確性

- [ ] 一次情報（公式発表・報道）を確認し、記事内にリンクした
- [ ] 事実・AI分析・筆者推論を明確に区別して表記した
- [ ] 未確認情報を記載していない

**判断例:**
> ✅ 「Anthropic公式ブログ（2025/3/5）によると〜」→ 一次情報あり、リンク済み
> ✅ 「ClaudeMixの見解では、この変化は〜を意味する可能性があります」→ AI分析と明記
> ❌ 「業界関係者によると近日中に〜が発表される予定らしい」→ 未確認情報は記載しない

---

## ✅ 読者への価値

- [ ] セクション2で「自分ごと化」できる視点を含んでいる
- [ ] セクション4に「今週中に取れるアクション」が1つ以上含まれている
- [ ] プロンプト例（セクション4）が実際に使えるレベルになっている

**判断例:**
> ✅ 「コンテキスト拡張により、長期プロジェクトの設計書全体をAIに渡せるようになります。あなたのワークフローで活かせそうな場面はありますか？」→ 自分ごと化◎
> ✅ 「今週中のアクション: Claude.aiの設定からコンテキスト上限を確認し、既存の長文プロンプトを再テストしてみてください。」→ 具体的◎
> ❌ 「ぜひ活用してみてください。」→ アクションが抽象的すぎる

---

## ✅ メタデータ

- [ ] カテゴリが `インフォメーション` になっている
- [ ] `isPaid: false` になっている
- [ ] タグに `"X"` `"トレンド"` ＋ 関連技術タグを含んでいる
- [ ] タイトルにシリーズ感がある（「AIに聞いてみた」「Claudeに聞いた」等）
- [ ] slugが `x-trend-[キーワード]` 形式になっている
- [ ] タイトルが30〜50文字に収まっている
- [ ] descriptionが80〜120文字に収まっている

**判断例:**
> ✅ tags: `["X", "トレンド", "Claude", "エージェント"]`
> ✅ slug: `x-trend-claude-4-launch`
> ❌ slug: `claude4-matome`（シリーズ規則に沿っていない）

---

## ✅ フォーマット

- [ ] ですます調で統一されている
- [ ] 見出しが `###` から `#####` の範囲で使われている
- [ ] コードブロックに言語指定がある
- [ ] 横線（`---`）はフロントマターとの区切りのみ
- [ ] Frontmatter（YAML）の構文が正しい

## チェックの使い方

1. すべての項目にチェックが入るまで修正する
2. 不明点は `docs/article-structure.md` と `docs/format-rules.md` を参照する
3. すべてOKになったらファイルを `results/[slug].md` に出力する
