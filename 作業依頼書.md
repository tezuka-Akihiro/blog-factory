# blog-factory 実装指示書 (Arch-Pattern Update)

## 1. アーキテクチャ方針

本プロジェクトは、提示された「コマンドアーキテクチャ」に準拠して実装します。

### レイヤー構造

```plaintext
src/
├── index.ts           # エントリーポイント（CommanderによるCLI定義）
├── commands/          # コマンド定義レイヤー
│   ├── micro/         # マイクロコマンド (scan, extract)
│   └── task/          # タスクコマンド (inspect)
├── tasks/             # ビジネスロジックレイヤー（純粋な関数）
│   ├── scan.ts        # ファイル探索ロジック
│   ├── extract.ts     # Frontmatter抽出ロジック
│   └── report.ts      # 結果書き出しロジック
├── types/             # 型定義 (index.ts)
└── utils/             # 共通ユーティリティ (logger.ts)
```

## 2. 実装タスク（Phase 1: Inspection）

### Step 1: 共通基盤の実装

- **src/utils/logger.ts**: 指定された絵文字プレフィックス（🔍, 📄, ✅, ⚠️, 💥）を持つロガーを実装。
- **src/index.ts**: `commander` ライブラリを導入し、コマンド登録の基盤を作成。

### Step 2: Micro Commands & Tasks の実装

- **Scan Task (src/tasks/scan.ts)**:
  - `BLOG_SOURCE_PATH` から記事ファイル一覧を返す純粋な関数。
- **Extract Task (src/tasks/extract.ts)**:
  - `gray-matter` を使用し、単一ファイルから `BlogPost` 型のデータを抽出する。
  - `title`, `category` の欠落を検知し、`Logger.warn` を出す。

### Step 3: Task Command の実装

- **inspect (src/commands/task/inspect.ts)**:
  - Scan -> Extract (指定カテゴリでFilter) -> Report (JSON保存) の一連のフローを実行。
  - `npm run inspect -- --category=XXX` で呼び出せるように `package.json` を構成。

## 3. Julesへの制約

- **ライブラリ**: `commander`, `gray-matter`, `dotenv` を使用。
- **Web標準**: ファイル操作は Node.js 標準の `fs/promises` を厳守。
- **品質**: 各 Task は、入出力が型定義（`src/types/index.ts`）に準拠した疎結合な関数として実装すること。
