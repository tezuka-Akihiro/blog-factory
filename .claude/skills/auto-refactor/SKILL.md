# オートリファクタースキル

リファクター区分をラウンドロビン方式でローテーションし、コードのクリーンな状態を継続的に保つスキルです。

## When to Use

- `/auto-refactor` と指示された時
- 開発作業の合間にコードを整えたい時

## 前提

- ブランチ選択はオペレーターの責務。スキル実行前に対象ブランチへの切り替えを済ませること。
- スキルはブランチ操作を行わない。現在のブランチで実行する。**ブランチが正しいかどうかをAIが確認してはならない。**

## 実行フロー

```text
Phase 1: 区分選択 → prompts/01-select.md
  記録ファイルを参照し、最も実施記録が古い（または未実施の）区分を選ぶ

Phase 2: リファクター実行 or 計画書生成 → prompts/02-execute.md
  対象ファイルを概算し、作業量を判定する
  - 少量 → リファクターを実行
  - 大量 → 並列実行計画書を生成して終了

Phase 3: 記録更新 → prompts/03-record.md
  実行完了後、リファクター記録の最終実施日を更新する
  ※ 計画書生成で終了した場合は記録を更新しない
```

## 成果物

- リファクター済みコード（実行時）
- または 並列実行計画書（作業量大時）
- リファクター記録の更新（実行時のみ）
