# 要件定義書：フェーズ2 信頼検証・リード獲得基盤

## 1. 目的

「生存戦略」に共鳴したユーザーが、一時的な訪問者から「継続的な読者（見込み客）」へ変化する過程を測定する。
指名検索や滞在時間を通じてブランドの浸透度を測り、有料会員化への転換率（CVR）を最適化するための判断材料とする。

---

## 2. 収集対象指標（KPI定義）

フェーズ1の指標に加え、以下の「質」に関わる指標を追加する。

| 指標名 | 取得元 | GA4 メトリクス / 取得方法 | ビジネス上の役割 |
| --- | --- | --- | --- |
| **指名検索数** | Google Search Console API | `searchanalytics.query` + `queryFilter: "ClaudeMix"` → `clicks` 合計 | 「ClaudeMix」等の単語による検索数。ブランド認知の証明。 |
| **平均エンゲージメント時間** | GA4 API | `averageEngagementTime`（expression: `userEngagementDuration/activeUsers`） | 重厚な記事がどれだけ深く読まれているか。 |
| **リード転換数（Micro-CV）** | Cloudflare Workers Analytics Engine | Workers Analytics Engine でカスタムイベント記録 | 無料レター登録・有料プランボタンのクリック等の具体的関心。 |
| **再訪率（Retention）** | GA4 API | dimension: `newVsReturning` × metric: `activeUsers` で比率計算（`returningUsersRate` は存在しない） | 一過性でなく、生活の一部（防衛拠点）として定着しているか。 |
| **平均読了率** | Cloudflare Workers Beacon | スクロール深度イベント（Workers Beacon 実装後） | スクロール深度を測定し、どの主張が読者の心を掴んでいるか特定。 |

---

## 3. 実装要件

### 3.1 取得手法

- **方式**: マルチソース集計。フェーズ1の蓄積データ（`results/kpi-history.json`）に加え、Google Search Console API・GA4 API からデータを統合
- **カスタムイベント計測**: サーバーサイド（Workers）で、特定ボタンクリックや PDF ダウンロード等を「イベント」として Workers Analytics Engine へ記録
- **認証**: 各 API の認証情報を `.env` で管理（下記参照）

```sh
# .env に追加する環境変数（フェーズ2）
GOOGLE_SERVICE_ACCOUNT_JSON=...   # Search Console / GA4 共通
GA4_PROPERTY_ID=...
```

> **事前確認済み（2026-03-06）**: Search Console・GA4 ともに**無料**で過去28日分のオンデマンド取得が可能。フェーズ1のような毎日蓄積が必須の制約はない。ただし下記のデータ遅延を考慮した日付指定が必要。
>
> | API | データ遅延 | 推奨 endDate |
> | --- | --- | --- |
> | Search Console | 通常 2〜3日 | `3daysAgo` |
> | GA4 | 24〜48時間（最大72h） | `2daysAgo` |

### 3.2 集計プロセス

1. **API統合（Fetch）**: `src/utils/search-console-client.ts`・`src/utils/ga4-client.ts` を新規作成し、同一期間のデータを一括取得
2. **信頼性スコアリング（Transform）**: 「指名検索数 × 再訪率」等の組み合わせで独自の「信頼スコア」を算出
3. **HTML生成（Render）**: フェーズ1の形式を維持しつつ、比較・分析用グラフを追加

### 3.3 型定義の拡張

`src/types/index.ts` の `ReportData.stats` に以下を追加する。

```typescript
brand: {
  namedSearchCount: number;   // 指名検索数（Search Console）
  avgEngagementTime: string;  // 平均エンゲージメント時間（GA4）
  returnRate: string;         // 再訪率（GA4）
  avgScrollDepth: string;     // 平均読了率（Workers Beacon）
};
conversion: {
  microCvCount: number;       // リード転換数（Workers Analytics Engine）
};
```

> **注意**: リード転換数（`microCvCount`）は Workers Analytics Engine への書き込みを claudemix 側（Astro / Workers）で実装した後に取得可能になる。未実装時はフォールバック値 `0` を返す。

---

## 4. レポート構成（HTML出力仕様）

コンシェルジュに「ファン化のプロセス」を証明するため、フェーズ1のレポートに以下のセクションを追加する。

- **ブランド・トラクション**: 指名検索キーワードの推移を時系列グラフで表示
- **エンゲージメント・マップ**: どの記事が最も長く読まれ、次にどのページへ遷移したかのパス解析
- **コンバージョン・ファネル**: 〔訪問〕→〔記事熟読〕→〔リード登録〕→〔有料プラン確認〕の離脱率を可視化
- **ターゲット適合度**: 主層（エンジニア）と副層（非技術者）どちらのコンテンツへの反応が良いかを比較

---

## 5. 運用要件

- **実行頻度**: 週次、または大きなコンテンツ投下後
- **実行コマンド**: `npm run report`（フェーズ1と同一）

### 判断基準（ネクストアクション）

| 観測結果 | 判断・対処 |
| --- | --- |
| 指名検索数が微増している | SNS での「思想発信」が成功していると判断 |
| 滞在時間が短い | 記事構造化（AI再学習用ナレッジとしての読みやすさ）を改善 |
| リード転換率が低い | 有料会員への導線（オファーの出し方）をコンシェルジュと再考 |
