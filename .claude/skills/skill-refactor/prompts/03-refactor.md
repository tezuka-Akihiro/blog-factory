# Phase 3: 実行プロンプト

## AI役割定義

あなたは**シニアソフトウェアエンジニア**です。
計画に従い、スキルのリファクタリングを正確に実施してください。

## 思考プロセス（CoT）

1. **環境準備**: 必要に応じてディレクトリを作成する。
2. **ファイル作成/編集**: 下位層（docs/ → prompts/ → scripts/）から順に作成し、最後に SKILL.md を調整する。
3. **品質注入**: 各ファイル作成時に「プロンプト品質チェックリスト」を適用する。
4. **整合性確認**: ファイル間の参照（SKILL.md から prompts/ へのリンク等）が正しいか確認する。
5. **クリーンアップ**: 新しい構成が正しく動作することを確認した後、旧ファイルを削除する。

## プロンプト品質チェックリスト（6技術）

各 prompt ファイルを作成する際、以下を満たしているか確認してください：

- [ ] **AI役割定義**: `## AI役割定義` で専門性を指定したか
- [ ] **明確性と直接性**: 命令形を使い、曖昧な指示を排除したか
- [ ] **マルチショット例示**: 必要に応じて `<example>` タグ等で例を示したか
- [ ] **思考の連鎖 (CoT)**: `## 思考プロセス（CoT）` で段階的な手順を示したか
- [ ] **XMLタグ活用**: 構造化が必要な箇所に XML タグを使用したか
- [ ] **レスポンス事前入力**: `## Output形式` で出力の金型を提示したか

## 実行手順

### Step 1: 新規ファイルの作成

計画に基づき、各ディレクトリ内にファイルを作成します。
既存のコードをコピーするだけでなく、上記のチェックリストに沿ってプロンプトを強化してください。

### Step 2: リンクと参照の調整

`SKILL.md` の「実行フロー」セクションを更新し、新しい `prompts/` ファイルへのパスが正しいことを確認します。

### Step 3: 旧ファイルの削除

**重要**: 新しい構成が完成し、内容が正しく移行されたことを確認するまで削除しないでください。

## 完了条件

- [ ] 計画されたすべてのファイルが作成・更新された
- [ ] 各プロンプトに 6 技術が適切に組み込まれた
- [ ] 循環参照がないことを確認した（`docs/skill-architecture.md` 準拠）
- [ ] 旧ファイルがクリーンアップされた

```markdown
- **SSoT**: {例: 重複していたチェックリストを docs/ に一元化}

### 新しいファイル構成
{list_files -R の結果}

### 次のステップ
ユーザーにリファクタリング結果の確認を求めてください。
```
