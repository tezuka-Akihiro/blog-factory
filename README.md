# blog-factory

「技術本 2.0」を製造・メンテナンスし続ける、構造的品質の司令塔。

## 概要

blog-factory は、本物の技術ドキュメント（Webサイト）の中身＝コンテンツを「製造ライン」として統制するためのリポジトリです。

単なる記事の保管場所ではなく、AIを「特定ドメインの専門家」として育て上げ、5〜10年「死なない」プロダクトを維持するための延命装置として機能します。

## 核心思想

- **最大の努力で最小の結果（死なないこと）**: 流行のブラックボックスを排し、Web標準に準拠した設計で、長期的な保守性を担保する。
- **AIを製造ラインとして統制**: AIを単なる執筆代行ではなく、構造的品質を守る「特定ドメインの作業員」として管理・教育する。
- **構造の不自然さを正す**: 既存記事の定期的な「サーチ（検診）」を通じて、情報の陳腐化や設計負債を早期に発見・修正する。

## 主な製造工程（ロードマップ）

### Phase 1: Inspection & Extraction (現在)

- **カテゴリ検索 (Search)**: 全記事から特定のカテゴリを指定し、タイトルと概要を高速に抽出。
- **棚卸し (Inventory)**: 既存資産の現状を把握し、AIが次に何をすべきかの判断材料を生成する。

### Phase 2: Assembly (将来)

- **AI執筆スキル (Writing)**: 蓄積されたナレッジを基に、AIがWeb標準（Remix）の型に沿った「本物のドキュメント」を下書き・校閲。

### Phase 3: Finishing & Packaging (将来)

- **プレビルド最適化**: Lighthouse 100点を維持するための画像処理、メタデータ、エッジ動的OGPの生成・検証。

## 技術スタック

「複雑性という悪」を排除するため、最小限かつ堅牢な構成を採用。

- **Language**: TypeScript
- **Runtime**: Node.js
- **Philosophy**: Web Standard First (Minimal Dependencies)

## 開発ガイド

### 初期セットアップ

```bash
npm install
```

### 記事の抽出（インスペクション）

特定のカテゴリに属する記事のタイトルと概要を抽出し、実行結果ファイルを出力します。

```bash
# 例：category「web-standard」の記事を抽出
npm run factory:inspect --category=web-standard
```
