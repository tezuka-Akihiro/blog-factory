# フォーマット規定

## 文字数

| 項目 | 規定値 |
| :--- | :--- |
| **記事全体** | 2,000〜3,500文字（速報性を優先） |
| **導入部** | 150〜250文字 |
| **セクション1** | 300〜500文字 |
| **セクション2** | 500〜800文字 |
| **セクション3** | 700〜1,200文字 |
| **セクション4** | 500〜800文字 |
| **タイトル** | 30〜50文字 |
| **description** | 80〜120文字 |

---

## タイトルフォーマット

シリーズ感（「〜をAIに聞いてみた」「Claudeに聞いた」等）を含める。

**✅ 良い例:**
```
Claude 4発表、バイブコーダーへの影響をAIに聞いてみた
EU AI Act施行、個人開発者は今すぐ何をすべきかClaudeに聞いた
話題のvibe coding、技術者として本当に警戒すべきことをClaudeに聞いた
```

**❌ NGパターン:**
```
最新AIツールについて詳しく解説します  → シリーズ感なし・何の話か不明
○○とは何か？完全解説              → 速報記事として重すぎる
○○についてまとめてみた             → シリーズ感がない
```

## slug フォーマット

```
x-trend-[キーワード]
```

- `x-trend-` を必ず先頭につける（シリーズ識別子）
- キーワードは英小文字・ハイフン区切り
- 意味が推測できる単語を使う

**✅ 良い例:**
```
x-trend-claude-4-launch
x-trend-eu-ai-act-enforcement
x-trend-vibe-coding
```

**❌ NGパターン:**
```
claude4-matome       → x-trend- プレフィックスがない
article-20250305     → 内容が推測できない
x-trend-             → キーワードがない
```

## frontmatter 全フィールド

```yaml
---
title: "[30〜50文字]"
description: "[80〜120文字]"
category: "インフォメーション"
tags: ["X", "トレンド", "[関連技術タグ1]", "[関連技術タグ2]"]
slug: "x-trend-[キーワード]"
author: "ClaudeMix Team"
publishedAt: "YYYY-MM-DD"
isPaid: false
---
```

**固定値:**
- `category`: `"インフォメーション"` 固定
- `author`: `"ClaudeMix Team"` 固定
- `isPaid`: `false` 固定
- タグの先頭2つ: `"X"` と `"トレンド"` 固定

## 見出しの使い方

- 本文の見出しは `###`（h3相当）から `#####`（h5相当）を使用
- `###` の見出しは記事全体で6〜8個程度に収める
- 各セクションのタイトルは `###` で統一

**例:**
```markdown
### [何が起きてる？] 状況の把握

#### OpenAIが「o3」を発表

...

### [自分たちは大丈夫？] 影響の分析
```

---

## 共通ルール

- **文体**: ですます調で統一
- **改行**: 文のまとまりで適度に改行を入れる
- **専門用語**: 初出時に説明を加える
- **コードブロック**: 言語指定を必ず追加（例: ` ```typescript `）
- **横線**: `---` はフロントマターとの区切りにのみ使用
- **リンク**: 必要最小限。一次情報源のリンクは明記する
- **断定表現**: 事実以外は断定しない（「〜と考えられる」「〜の可能性がある」）
