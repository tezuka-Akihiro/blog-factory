# Phase 1: 入力検証

オペレーターが `$ARGUMENTS` として渡したトレンド情報を評価する。
**不足があれば執筆を開始せず、補足を求めて停止すること。**

## 評価手順

以下の3指標をすべて満たしているか確認する。

| 指標 | 判定条件 |
| :--- | :--- |
| **トレンドの核心** | 「何が」「どう変わった/起きた」が1文で言える |
| **一次情報の所在** | URL・発表元・報道日時のいずれかが含まれている |
| **読者への接点** | バイブコーダー・技術者・個人事業主への影響が想像できる |

## 判定と対応

### ✅ 3指標すべて満たす → Phase 2 へ進む

入力例（十分）:

> 「Anthropicが Claude 4 を発表。コンテキスト200万トークン、エージェント機能が大幅強化。公式: https://... バイブコーダーの開発フローへの影響を書いてほしい。」

> 「EU AI Act の施行細則が確定。オープンソースモデルの扱いが焦点。TechCrunch記事あり。個人事業主が今すぐ知るべきことをまとめてほしい。」

### ❌ 不足がある → 補足を求めて停止

不足している指標に応じて、以下のように返す。

**トレンドの核心が不明な場合:**
> 「どのような発表・出来事でしょうか？何が変わった/起きたのかを1文で教えてください。」

**一次情報が不明な場合:**
> 「公式発表や報道記事のURLがあれば共有してください。出典がないと事実と推測の区別が難しくなります。」

**読者への接点が見えない場合:**
> 「このトレンドは、バイブコーダーや個人事業主にどう関係しますか？記事の方向性を教えてください。」

入力例（不足）:

> 「Gemini が話題になってる。記事書いて。」
> → 返答: 「どのような発表・出来事でしょうか？公式発表や報道のURLがあれば共有してください。」

## 完了条件

3指標をすべて満たしていることを確認したら、以下を簡潔にまとめてオペレーターに提示し、次のフェーズへ進む。

```
【入力確認】
- トレンドの核心: [1文でまとめた内容]
- 一次情報: [URLまたは発表元]
- 記事の方向性: [読者への接点]

→ Phase 2（AI分析）に進みます。
```
