# 要件定義書：フェーズ1 マーケティング計測基盤（ローカル集計版）

## 1. 目的

会員数0〜10名の初期段階において、サービスの「生存」と「市場の初動」を客観的に測定する。
外部タグを排除した軽量なWeb標準構成を維持しつつ、コンシェルジュとの面談に耐えうるビジネスレポートをAPI経由で自動生成する。

---

## 2. 収集対象指標（KPI定義）

フェーズ1においてビジネス判断に必要な指標を以下の5点に定義する。

| 指標名 | 取得元（Cloudflare API） | ビジネス上の役割 |
| --- | --- | --- |
| **UU（Visits）** | `visits` | 認知の総量。市場に「発見」されているかの確認。 |
| **PV（PageViews）** | `pageViews` | 興味の深さ。一人あたり何記事読んでいるか。 |
| **流入経路** | `referrerHost` | どの媒体（SNS/検索）に注力すべきかの判断材料。 |
| **人気記事** | `requestPath` | どの「生存戦略」がターゲットに響いているかの特定。 |
| **システム健全性** | `edgeResponseStatus` | エラー率の監視。構造的欠陥による機会損失の防止。 |

---

## 3. 実装要件

### 3.1 取得手法

- **方式**: Cloudflare GraphQL Analytics API を用いたデータ抽出
- **データセット**: `httpRequestsAdaptiveGroups`（サンプリングなしの正確な数値）
- **認証**: Cloudflare API Token によるセキュアな通信

### 3.2 集計プロセス

1. **データ取得（Fetch）**: TypeScript スクリプト（`src/utils/cloudflare-client.ts`）で GraphQL エンドポイントへリクエスト
2. **構造化（Transform）**: 取得した JSON データをビジネスサマリー用のオブジェクトへ変換
3. **HTML生成（Render）**: ローカルテンプレートへデータを注入し、HTMLファイルを生成

### 3.3 型定義の拡張

`src/types/index.ts` の `ReportData.stats.traffic` に以下フィールドを追加する。

```typescript
traffic: {
  pv: number | string;
  uu: number | string;
  avgStayTime: number | string; // フェーズ1では取得対象外。常に '-' を返す
  topSources: Array<{ host: string; visits: number }>; // 流入経路TOP
  topPages: Array<{ path: string; requests: number }>; // 人気記事TOP 5
  errorRate: string; // edgeResponseStatus から算出。stats.monitoring.errorRate へ上書き
}
```

> **注意**: `avgStayTime` は Cloudflare Analytics では直接取得できないため、フェーズ1の計測対象外とし `'-'` 固定とする。
>
> **注意**: `edgeResponseStatus` から算出したエラー率は `stats.monitoring.errorRate` フィールドに格納する（現在のハードコード値 `"0.0%"` を実値に差し替える）。

---

## 4. レポート構成（HTML出力仕様）

コンシェルジュに「事業の進捗」を直感的に伝えるため、以下のレイアウトとする。

- **ヘッダー**: 生成日時、計測期間（直近24時間 / 7日間）
- **メインメトリクス**: UU・PV・エラー率をカード形式で大きく表示
- **チャート / リスト**:
  - 流入ドメイン分布: X（Twitter）・検索エンジン・直接流入の比率
  - 人気コンテンツ TOP 5: アクセスの多いURLパスを降順で表示
- **ビジネス・ステータス**: 「会員数0→1への進捗」をメモできるフリースペース

---

## 5. 運用要件

- **実行環境**: ローカル開発環境のコマンドライン（`npm run report`）
- **保存**: 生成された HTML は `results/` ディレクトリに出力
- **活用**: 週に一度、生成された HTML をコンシェルジュに提示し、流入経路の変化に基づいたターゲット（主層/副層）の微調整を相談する
